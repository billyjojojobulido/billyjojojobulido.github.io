Data Structure and Algorithm - Array(IV): Divide & Conquer
August 09, 2024
5k words 20 mins

Overview

"Divide and conquer" literally means to "divide and manage," which involves breaking a complex problem into two or more identical or similar sub-problems, until the sub-problems are simple enough to be solved directly. The solution to the original problem is the combination of the solutions to the sub-problems.

Process of Divide and Conquer

The core idea of the divide-and-conquer algorithm is "divide and conquer."

Therefore, the general process of divide and conquer can be broken down into three steps: Divide -> Conquer -> Combine

  1. Divide the original problem into smaller, structurally similar sub-problems.
  2. Solve the sub-problems recursively until reaching a base case that can be solved directly.
  3. Combine the solutions of the sub-problems to form the solution to the original problem.

Features Suitable for Divide and Conquer

By reverse-engineering the process of divide and conquer, we can identify the characteristics that a problem should have to be suitable for this approach:

  1. The problem becomes easier to solve when reduced to a certain size.
  2. The problem can be divided into smaller, identical problems (optimal substructure property). The solution to the sub-problems can be combined to solve the overall problem.
  3. The sub-problems are independent of each other.

Given these features, it is also important to recognize when to avoid using divide and conquer:

  • When the problem has high dimensions

    • This feature violates both Principle 1 and Principle 3. For example, in graph connectivity problems, if the problem involves high dimensions or multilayered relationships, it becomes challenging to decompose the problem into independent sub-problems.

  • When the problem involves global optimization

    • This feature violates Principle 2. For example, in the Traveling Salesman Problem (TSP), where the goal is to find the shortest path that visits all cities, the problem requires considering all possible paths. In this case, no matter how the problem is divided, the solution to each sub-problem can impact the global solution, leading to local optimizations that disrupt global consistency and make it difficult to combine the solutions into a global optimum.

  • When sub-problems have strong dependencies

    • This feature violates Principle 3. For example, in the 0/1 Knapsack Problem, although the problem can be divided into sub-tasks (e.g., considering whether to include an item in the knapsack), the solution to each sub-problem depends on the solutions to other sub-problems (whether other items are included in the knapsack). Such problems are better solved using dynamic programming.

These problems often require other algorithms and strategies, such as backtracking, greedy algorithms, dynamic programming, heuristic algorithms, etc. When solving these problems, it is important to consider the overall structure and characteristics of the problem, rather than just decomposing it.

Merge Sort Using Divide and Conquer

Merge Sort is perhaps the most classic example of a divide-and-conquer algorithm. The general divide-and-conquer approach is demonstrated in the code below:

def merge_sort(array):
    if (solvable directly) return;
    merge_sort(left half of array);
    merge_sort(right half of array);
    merge(left half of array, right half of array);

Since the divide-and-conquer pattern is Divide -> Conquer (reach base case) -> Combine (backtrack), the left and right halves are divided first, then solved, and finally combined during backtracking, which is similar to post-order traversal.

Decrease-and-Conquer

The "decrease-and-conquer" approach is a special case of the divide-and-conquer approach: at each step, the problem is transformed into a smaller sub-problem. The "decrease-and-conquer" approach is typically applied in algorithms like "binary search," "selection sort," "insertion sort," and "quick sort." The difference between "divide-and-conquer" and "decrease-and-conquer" is as follows:

  • Divide-and-Conquer: The problem is divided into several sub-problems, which are then solved individually. The solutions to these sub-problems are combined to form the solution to the original problem.
  • Decrease-and-Conquer: When dividing the problem, it is transformed into a single, smaller sub-problem. Therefore, the solution to the sub-problem is the solution to the original problem, with no need for a "combine" step.

Recursion and Divide and Conquer

Since recursion and divide-and-conquer are often intertwined, many beginners struggle to differentiate between the two. However, after some practice, the differences become clear:

  • Recursion is a programming technique, a way of thinking about solving problems.
  • Divide-and-Conquer is largely based on recursion, and is a problem-solving algorithm.
